# HobbyLedger Roadmap

这个文档用于记录 `HobbyLedger` 后续版本的主要演进方向。

当前项目已经完成 `v1 基础版` 的核心闭环。下面的内容不是严格的发布时间承诺，而是当前比较明确、值得持续推进的产品路线。

## 当前阶段

当前版本已经具备：

- 本机记账
- 按爱好分类管理支出
- 首页汇总统计
- 分类详情查看
- 基础测试覆盖

接下来的重点，不再是“能不能记账”，而是把这个产品往更长期、更完整、更有洞察力的方向推进。

## Roadmap Themes

### 1. UI 与视觉优化

目标：

- 提升整体观感和品牌感
- 让首页统计更清晰、更有层次
- 让记账流程更顺手，减少操作阻力

可能包含：

- 更完整的颜色系统与视觉规范
- 更统一的卡片、列表和图表样式
- 更好的空状态、引导页和启动页
- 更完整的 App Icon 和品牌元素

价值：

- 提高日常使用意愿
- 让产品从“能用”提升到“愿意长期打开”

### 2. 云端同步与跨设备支持

目标：

- 支持数据云端共享
- 支持在多设备之间无缝查看和录入

计划方向：

- iPhone / iPad / MacBook 之间同步
- 未来探索 Apple Watch 场景
- 保证多端数据一致性和同步稳定性

可能涉及：

- iCloud / CloudKit
- 多端布局适配
- 同步冲突处理与恢复机制

价值：

- 让这个账本不再局限于单机
- 让用户在不同设备上都能持续使用同一套数据

### 3. 更细化的记账体系

目标：

- 让记账不只停留在“爱好一级分类”
- 支持更细的消费结构分析

计划方向：

- 增加更多子类
- 支持爱好下的二级分类或更细粒度分类
- 例如摄影可拆分为：
  - `数码`
  - `胶片`
  - `镜头`
  - `胶卷`
  - `冲扫`

价值：

- 帮助用户真正看清“钱到底花在了爱好的哪个部分”
- 为后续统计和 AI 分析提供更细的数据基础

### 4. 月度 / 年度消费可视化

目标：

- 从“累计花了多少”扩展到“某个时间段花了多少”

计划方向：

- 月度消费视图
- 年度消费视图
- 不同爱好在月度 / 年度维度下的分布

可能包含：

- 柱状图
- 折线图
- 时间筛选器
- 年/月切换

价值：

- 帮助用户理解长期消费节奏
- 更容易看出季节性消费和周期性支出

### 5. 消费趋势与历史对比

目标：

- 不只是展示静态数据，还能展示变化趋势

计划方向：

- 本月 vs 上月
- 今年 vs 去年
- 某个爱好最近几个月的变化
- 某个周期的增长 / 下降趋势

价值：

- 让用户知道自己是不是花得越来越多
- 帮助形成更长期的消费认知

### 6. AI 驱动的数据分析

目标：

- 在数据积累后，引入更智能的分析能力

可能方向：

- 自动总结最近消费特点
- 识别某些爱好的异常增长
- 生成阶段性消费观察
- 提示某类消费是否明显高于过去平均水平
- 帮助用户发现自己没注意到的消费模式

说明：

- 这一部分应建立在前面几个阶段完成之后
- AI 能力依赖更完整、更干净的数据结构

价值：

- 从“记账工具”升级到“消费洞察工具”

## Suggested Evolution Order

结合当前项目状态，建议的推进顺序是：

1. UI 与视觉优化
2. 更细化的记账体系
3. 月度 / 年度消费可视化
4. 消费趋势与历史对比
5. 云端同步与跨设备支持
6. AI 驱动的数据分析

这样安排的原因是：

- 先把当前单机产品体验做扎实
- 再把数据结构做细
- 再在更细的数据基础上做图表与趋势
- 最后再处理同步和 AI 这种更高复杂度能力

## Notes

目前这份路线图只记录了已经明确想到的方向。

后续如果出现新的想法，建议继续补充到这个文档，例如：

- 预算体系
- 标签系统
- 导出能力
- 退款 / 二手回血记录
- 消费提醒或复盘机制

这个项目本身就是一个会不断成长的产品，所以路线图不需要一次写完，但应该持续更新。
